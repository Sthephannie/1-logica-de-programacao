MÓDULO 1: INTRODUÇÃO AO DESENVOLVIMENTO E LÓGICA DE PROGRAMAÇÃO 1.1 O que é programação O objetivo da programação é de desenvolver software para outras pessoas. Desenvolver software é resolver problemas usando programação como ferramenta. Toda empresa hoje ou é ou ainda não percebeu que é digital, então existe uma necessidade de desenvolver sites, apps e soluções para essas empresas. O programador precisa entender a necessidade de um usuário e transformar essa necessidade em uma aplicação, em um software, em um sistema. O software só é software se ajuda alguém. Caso contrário, só é um amontado de instruções. As ferramentas necessárias para desenvolver essas aplicações são: • linguagens de programação (para conversar no idioma do computador), • banco de dados (para armazenar a informação), • WWW (para comunicação), • apresentação de informação - front-end (identidade visual, interface, interação, intuitivo). 1.2 Dados do mercado Em março de 2021, tínhamos 20 mil vagas de desenvolvimento abertas em SP, desde júniores a arquitetos de sistema. O mercado está muito aquecido e temos uma defasagem em relação à demanda, visto que muitas empresas precisam de soluções digitais. O mercado de desenvolvimento de aplicações pode atender empresas tradicionais, que estão migrando as operações para o digital (ex: Itau). E também as empresas que já nasceram digitais (ex: Nubank). É possível atender do micro até as grandes empresas. Algumas áreas no mercado de programação: • Desenvolvedores, • Analistas de infraestrutura, • Analistas de Requisitos, • Testadores, • Profissionais de UI/UX, • Profissionais de Telecomunicações, • Profissionais de DevOps. Também é possível trabalhar com empresas nacionais e até internacionais de forma remota. 1.3 Como é a carreira de desenvolvimento Hard skills • Back-end: softwares que fazem a manipulação de dados do sistema ex de e-commerce: selecionar o produto, efetuar o pedido, gravar o endereço na base de dados, enviar o pedido para a equipe de logística) • Front-end: está relacionado com a interação com o usuário ex: navegação, interface, apresentação da informação, facilidade de uso, intuitividade para chegar na informação • Aplicativos: aplicativos nativos ou não • Base de dados: armazenamento e recuperação de informação • GIT e versinamento • Framework de desenvolvimento ágil Soft skills • Trabalhar em grupo: prazos curtos, projetos grandes • Liderança Carreira em T • Junior: sabe um pouco de front, back, fez um site simples, não fez integração de sistemas, trabalhou um pouco com GIT e agile • Pleno: começa a se envolver em problemas mais complexos e evoluem suas skills técnicas em uma competência 1.4 Linguagens, frameworks e bibliotecas • Linguagem de programação: idioma em que eu quero conversar com o computador – ex: Java, PHP, Javascript, C+, Phyton • Linguagem de marcação – ex: HTML • Linguagem de estilização – ex: CSS • Bibliotecas: pequenos trechos de código que eu reaproveito ex: estrutura de pedidos que eu utilizo no sistema de vendas, de estoques, de relatórios para o gestor / estrutura que utilizo no desktop do site e no mobile) • Frameworks: bibliotecas com manual de uso, que agilizam e tornam mais seguro o desenvolvimento ex: do front-end (Angular, ReactJs) 1.5 Editores de texto • Editores de texto: Visual Studio Code • IDE (Ambientes Integrados de Desenvolvimento): toda IDE também é um editor de texto, ele já corrige o código em tempo real, melhoria ex: Eclipse IDE
